Les paramètres d'URLs utiles:

- Afficher la carte G-Loc :
http://ledudulela.free.fr/gloc
http://ledudulela.free.fr/gloc/?i=999


- Les applications émettrices de positions géographiques ( ex: GPSLogger ) :
http://ledudulela.free.fr/gloc/?lat=%LAT&lon=%LON&i=999
http://ledudulela.free.fr/gloc/?lat=%LAT&lon=%LON&i=999&u=alias

   Arguments implémentés:
  • i=999 (obligatoire)
  • u=alias (doit appartenir à la liste des alias prédéfinie)
  • lat=%LAT (obligatoire)
  • lon=%LON (obligatoire)
  • time=%TIME (voir doc application)
  • alt=%ALT (altitude)
  • spd=%SPD (speed)
  • desc=%DESC (description)
  • batt=%BATT (battery)
  • acc=%ACC (accuracy, precision)
  • all (tous, uniquement pour Overland sur iPhone)
Remarques:
Pour Overland, il est inutile de mentionner les arguments lat et lon et il est inutile de définir la valeur des arguments optionnels %VAL .
Exemple: url_serveur/gloc/?i=999&u=alias&spd&batt


Pour Osmand (greffon d'enregistrement, option Live), les valeurs des arguments lat et lon sont respectivement {0} et {1}
Exemple: url_serveur/gloc/?i=999&u=alias&lat={0}&lon={1}

Problèmes habituellement rencontrés:
Si gpslogger n'affiche pas les coordonnées alors que le service de localisation est activé, redémarrer le smartphone.
Si gpslogger affiche les coordonnées mais que gloc ne les affiche pas, attendre quelques secondes et actualiser la page.
Si les coordonnées ne s'affichent toujours pas sur gloc:
- S'assurer que le smartphone de l'émetteur a une connexion internet (data activée)
- S'assurer que l'émetteur a correctement configuré l'URL d'enregistrement dans gpslogger (i & u)
- S'assurer que l'émetteur a activé l'envoi automatique par Custom URL


Economiser la batterie:
Utilisation du service de localisation réseau (téléphonique) plutôt que satellite.
À configurer dans les paramètres du smartphone ET dans les paramètres de gpslogger.
Cette configuration peut également résoudre le problème d'obtention de position géographique de gpslogger.
smartphonegpslogger


Enregistrer la trace:
L'application gpslogger permet d'enregistrer le trace dans un fichier (GPX, KML, NMEA)

Des sites internet permettent ensuite d'afficher la trace (fichier gpx) sur une carte.
Exemple: www.calculitineraires.fr

Applications compatibles:
  • Android - Osmand+
  • Android - Osmand (avec un abonnement de quelques centimes pour l'option Live)
  • Android - gpslogger
  • Android - phonetrack
  • iPhone - Osmand (avec un abonnement de quelques centimes pour l'option Live)
  • iPhone - Overland GPS Tracker
Pour Android, les applications sont disponibles dans l'application ou site internet F-Droid .
L'application Osmand est disponible sur google playstore et sur apple store.
